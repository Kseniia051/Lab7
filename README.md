## Ответы на вопросы

### 1. Какие jQuery-функции использовались?

- `$(document).ready()` — ожидание полной загрузки DOM.  
- `$()` — универсальный селектор элементов.  
- `.click()` — обработка клика по элементу.  
- `.addClass() / .removeClass()` — добавление и удаление CSS-классов.  
- `.data()` — получение значения data-атрибута.  
- `.animate()` — плавная анимация прокрутки страницы.  
- `$.ajax()` — отправка GET-запроса для загрузки цитаты.  
- `.draggable()` — jQuery UI, делаем элемент перетаскиваемым.  
- `.datepicker()` — jQuery UI, превращаем input в календарь.  

---

### 2. Что показалось быстрее/проще — jQuery или чистый JS?

- **jQuery** проще и короче писать, особенно для **обработки событий, селекторов и анимаций**.  
- Для **современного JS (Vanilla JS)** многое делается через `querySelector`, `classList`, `addEventListener`, `fetch` — немного длиннее, но без сторонних библиотек.  
- jQuery был полезен раньше для **кросс-браузерности**, сейчас большинство браузеров поддерживают стандарты, и Vanilla JS часто быстрее и легче для загрузки.  

---

### 3. Что из этого вы бы оставили, если писали современный сайт?

- **Vanilla JS** для всех основных интерактивных функций (кнопки, вкладки, прокрутка, AJAX).  
- **jQuery UI** или готовые библиотеки можно оставить для сложных UI-компонентов (drag & drop, datepicker), если не хочется писать всё с нуля.  
- Основной принцип: использовать **легковесные решения и современные стандарты**, jQuery оставить только для поддержки старого кода или существующих плагинов.
